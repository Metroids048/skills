## §4.5 Mini-Spec（结构化任务单）

```yaml
# --- 以下由 requirement-clarifier 填写；用户确认前不得执行 ---
meta:
  原始需求: "<用户原话一句话>"
  需求类型: "<写代码|前端Demo|UI/UX|混合|…>"
  当前阶段: "<想法|原型|初版后优化|…>"
  消息分型: "B 模糊实施"

用户与场景:
  目标用户: "<谁>"
  使用场景: "<何时何地>"
  现状痛点: "<1-3 条>"

目标与验收:
  核心目标: "<一句话>"
  成功标准:  # 必须可观察
    - "<验收 1>"
    - "<验收 2>"
  不做什么:  # 硬性边界
    - "<不做 1>"

范围:
  涉及页面/模块: ["<path 或页面名>"]
  涉及文件: ["<预计文件>"]  # 待确认可标 (待确认)
  是否新模块: "<是|否|待确认>"
  MVP第一刀: "<若只做一步，做什么>"

约束:
  技术栈: "<如 prototype 静态 HTML/JS>"
  设计输入: "<Figma|截图|无|待确认>"
  项目门禁: ["<如 verify-all, ADR-003>"]
  依赖其他任务: "<无|…>"

intent_restate:  # 来自 interview-protocol（必填）
  outcome: "<可观察结果>"
  user: "<谁受益>"
  success: "<怎样算做完>"
  out_of_scope: "<明确不做>"

待确认:  # ≤5，影响方向
  - "<问题 1>"

默认建议:  # 用户不回复时的假设（idea-refine 风格 Not Doing）
  - "<若无特别说明则…>"

assumptions_audit:  # clarify-first 风格
  weighted_total: "<N>"
  critical_weight2: ["<环境|依赖|跨文件>"]
  risk_level: "<LOW|MEDIUM|HIGH>"
  phase: "<clarify-only|plan-confirmed|execute>"

风险:
  - "<范围失控|导航回归|…>"
```

**给用户的一句话摘要（必填，≤3 行 plain 中文）：**

> 我理解你想做的是：…。第一刀建议先 …。需要你确认的关键点是：…。
